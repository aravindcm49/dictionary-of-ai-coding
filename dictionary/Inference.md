Running a trained [[Model]] to generate output — what happens on every model provider request (see [[Model provider request]]). Parameters (see [[Parameters]]) stay fixed; the model just does next-token prediction (see [[Next-token prediction]]) over the context (see [[Context]]) it's given. Cheap relative to training (see [[Training]]), but billed per token (see [[Token]]) and the dominant cost of using a model.

*Usage:* "Why does the bill scale with usage instead of being a flat license?" "You're paying for inference — every model provider request runs the model on the provider's hardware. Training already happened, but inference costs accrue per request, and a single turn can fan out into many requests when tools are called."
