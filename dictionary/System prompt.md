The instructions the harness (see [[Harness]]) prepends to every model provider request (see [[Model provider request]]) — the agent (see [[Agent]])'s standing brief: who it is, how to behave, which tools (see [[Tool]]) it can call, what conventions to follow. Usually stable across a session (see [[Session]]).

*Usage:* "Two harnesses, same model, totally different behavior on the same prompt." "Different system prompts. One's tuned for terse code edits, the other for explaining — that's where the divergence lives, before your message even arrives."
