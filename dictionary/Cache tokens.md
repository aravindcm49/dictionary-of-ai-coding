Input tokens (see [[Input tokens]]) the provider (see [[Model provider]]) has cached from a previous model provider request (see [[Model provider request]]) so it doesn't have to re-process them. When consecutive requests share a prefix, the provider reuses the work and bills the cached portion at a much lower rate. The lever that makes long sessions (see [[Session]]) affordable — without it, every turn (see [[Turn]]) re-pays for the whole history.

*Usage:* "Cost on long sessions is brutal — eight bucks for a refactor." "Check the cache tokens. If the harness is reordering the system prompt or files between turns, the prefix breaks and you re-pay full input rate every request."
